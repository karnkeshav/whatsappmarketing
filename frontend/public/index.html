Process

How it works.

Fully GitHub-native — no servers, no database, no paid APIs. The bot runs in GitHub Actions, data lives in a JSON file in this repo, the site is GitHub Pages.

⌕
01 · Discover

Pick a state or capital → a GitHub Action runs the bot for that location and appends new groups to data/groups.json.

◉
02 · Render

Your browser fetches the raw JSON from this repo and renders it. No backend, no API.

⚑
03 · Crowd-verify

Buttons on every card open pre-filled GitHub Issues: Worked, Needed approval, Only admins post, Invalid.

⌗
04 · Auto-flag

2× "approval" → hidden. 2× "only admins post" → READ-ONLY badge but stays visible so you can DM the admin.
