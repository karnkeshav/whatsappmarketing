Process

How it works.

A 100% GitHub-native stack. No servers, no database, no paid APIs. The bot runs in GitHub Actions, the data lives in a JSON file in this repo, and the site is GitHub Pages.

⌕
01 · Discover

A GitHub Action runs daily, searches the public web for chat.whatsapp.com invite links + jobs + region, and commits new ones to data/groups.json.

◉
02 · Render

Your browser fetches the raw JSON from this repo and renders it. No backend, no API, no auth needed.

⚑
03 · Crowd-verify

Visitors click "Worked / Needed approval / Invalid" — buttons open a pre-filled GitHub Issue. Another Action tallies the issues.

⌗
04 · Auto-hide

Two "needed approval" reports → the group is removed from the JSON. Two "works" reports → it gets a verified badge.
